# Инструкции для Claude Code

## Структура проекта

### Папка DEV
Все тестовые и вспомогательные скрипты должны находиться в папке `dev/`:
- Тестовые файлы (test_*.py, check_*.py)
- Скрипты настройки и конфигурации (setup_*.py)
- Утилиты для разработки (update_*.py, reactivate_*.py, add_*.py)
- Любые временные или экспериментальные файлы

### Папка sessions
Все файлы сессий Telegram (*.session, *.session-journal) должны храниться в папке `sessions/`:
- Файлы создаются автоматически при авторизации
- Папка исключена из git через .gitignore
- В коде используются пути вида `sessions/имя_сессии`

## Правила создания файлов

### ВАЖНО: Размещение тестовых файлов
- При создании новых тестовых файлов ВСЕГДА размещайте их в папке `dev/`
- НЕ создавайте тестовые файлы в корневой директории проекта
- Примеры тестовых файлов:
  - `dev/test_*.py` - тесты функциональности
  - `dev/check_*.py` - проверки и диагностика
  - `dev/setup_*.py` - скрипты настройки
  - `dev/*_test.py` - альтернативный формат тестов

### Исключения
В корне проекта должны находиться только:
- `main.py` - основной файл запуска
- `config.py` - конфигурация проекта
- Модули производственного кода (bot/, parsers/, data/, utils/, services/)
- Документация (README.md, SETUP.md и т.д.)

## Работа с сессиями Telegram
- Все новые сессии создаются с префиксом пути `sessions/`
- Пример: `sessions/tg_session_12345_79001234567`
- Не храните файлы сессий в корне проекта
- При миграции старых сессий перемещайте их в `sessions/`
